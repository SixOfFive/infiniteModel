# InfiniteModel dependencies.
#
# Install only what a given machine needs:
#
#   Controller (server.py):   pip install fastapi uvicorn
#   Worker, Milestone 1:      pip install psutil
#   Worker, Milestone 2+:     pip install psutil torch transformers safetensors huggingface_hub
#
# torch is large and its wheels lag new Python releases — install it on the
# workers only when model execution lands (M2). M1 (registry/dashboard) needs
# just the controller + psutil stack below.

# --- Controller (server.py) ---
fastapi
uvicorn

# --- Controller (server.py) — Multimodal (vision + audio), CONTROLLER-SIDE only ---
# Image/audio models (Qwen2.5-VL, Gemma-4 vision unified+tower, Pixtral/Mistral3, Qwen2.5-Omni)
# decode + preprocess on the CONTROLLER, not the workers. Without pillow EVERY vision request
# ImportErrors ("No module named 'PIL'") — and transformers caches the availability check at
# import, so install BEFORE starting the controller (or restart it after). torchvision is
# deliberately NOT required: every image processor here has a PIL-only backend
# (Qwen2VLImageProcessorPil / Gemma4ImageProcessorPil / PixtralImageProcessorPil), and adding
# torchvision risks pip pulling a torch that doesn't match the node's pinned build.
pillow            # images — required for ANY vision model
# Audio-in decode: soundfile (WAV/FLAC/OGG; needs the libsndfile system lib) then librosa
# (adds mp3 + resample, heavier — pulls numba which pins numpy). BOTH optional: audio-in falls
# back to stdlib `wave` for PCM WAV, and TTS/speech-out already writes WAV via stdlib `wave`.
soundfile         # optional — audio-in for WAV/FLAC/OGG
# librosa         # optional — audio-in for mp3 + high-quality resample

# --- Worker (client.py) — Milestone 1 ---
psutil

# --- Worker (client.py) — Milestone 2+ (model loading & execution) ---
# torch
# transformers
# safetensors
# huggingface_hub
# einops         # required by some models' trust_remote_code (e.g. nomic-embed-text encoder)
# nvidia-ml-py   # optional, only for VRAM reporting on CUDA nodes
#
# AMD GPU (ROCm) worker: do NOT use a CPU/CUDA torch — install a ROCm build matched to
# your GPU arch. For Strix Halo (gfx1151) the generic download.pytorch.org wheels crash
# in the bundled HSA runtime; use AMD's arch-specific TheRock wheels instead. The helper
#   ./install-rocm.sh [arch]      # default gfx1151
# does the whole venv (torch + matched ROCm runtime + these deps). See docs/ROCM.md.
